# СКМ: сайт и MVP-кабинет

Next.js 15 App Router + TypeScript + Tailwind CSS + shadcn-style UI + Framer Motion + Lenis + SQLite.

## Запуск

```bash
npm install
npm run dev
npm run build
npm run lint
```

Локально сайт открывается на `http://127.0.0.1:3000`.

## Как задать admin email

Админка скрыта по адресу `/admin`. Ссылки на нее нет в обычной навигации.

Основная настройка находится в `lib/constants.ts`:

```ts
export const ADMIN_EMAIL = (process.env.ADMIN_EMAIL ?? "admin@skm.ru").trim().toLowerCase();
```

Чтобы поменять администратора в коде, замените `admin@skm.ru` в `lib/constants.ts`.
Чтобы поменять администратора без правки кода, задайте переменную в `.env.local`:

```env
ADMIN_EMAIL=owner@skm.ru
```

Доступ к `/admin` получает только пользователь, вошедший через OTP с этим email. Остальным пользователям страница показывает сообщение "Доступ запрещён".

`AUTH_SECRET` тоже лучше задать в `.env.local`, чтобы подпись сессии не зависела от dev-значения:

```env
AUTH_SECRET=long-random-secret
```

Если `ADMIN_EMAIL` не задан, dev-значение по умолчанию: `admin@skm.ru`. Даже если в старой SQLite-базе у пользователя осталась роль `admin`, effective role пересчитывается по email.

## Как заменить логотип

Header использует файл из `siteConfig.logoPath`.

По умолчанию:

```ts
// lib/site-config.ts
logoPath: "/logo.png"
```

Чтобы заменить логотип, положите новый файл в `public/logo.png`. Если хотите использовать SVG, положите `public/logo.svg` и обновите `logoPath` и `src` в `components/header.tsx`.

## Как работает вход по OTP

Пользователь вводит email, сервер создает 6-значный код, хранит в SQLite только HMAC-хеш и срок действия. Код живет 15 минут, является одноразовым и после успешной проверки помечается использованным. Повторный запрос ограничен: не чаще одного раза в 60 секунд и не больше 5 запросов за час на один email.

Коды отправляются через Resend. Если `RESEND_API_KEY` не настроен в dev-режиме, сайт покажет dev-код на экране и выведет его в консоль сервера. В production без ключа отправка вернет ошибку.

## Как подключить реальную отправку email через Resend

Точка подключения вынесена в `lib/resend.ts` и `lib/email.ts`.

1. Установите пакет:

```bash
npm install resend
```

2. Получите API key:

- Зайдите на [resend.com](https://resend.com)
- Откройте `API Keys`
- Создайте ключ с правом отправки email
- Скопируйте ключ, обычно он начинается с `re_`

3. Добавьте переменные в `.env.local`:

```env
RESEND_API_KEY=re_...
OTP_FROM_EMAIL=SKM <no-reply@your-domain.ru>
```

Для быстрой проверки можно оставить `OTP_FROM_EMAIL=SKM <onboarding@resend.dev>`, но для реального домена лучше подтвердить домен в Resend и указать свой отправитель.

4. Перезапустите dev-сервер:

```bash
npm run dev
```

## Баланс и оплаты

- Форма "Оставить заявку" на главной странице сохраняет обращения в `contact_requests`.
- В `/admin` секция "Заявки" позволяет просматривать обращения, менять статус `new` -> `in_progress` -> `processed` и удалять заявки.
- Пополнение баланса больше не начисляется мгновенно.
- Кнопка "Запросить пополнение баланса" открывает форму суммы и создает запись в `topup_requests` со статусом `pending`.
- В `/admin` есть секция "Запросы на пополнение" со счетчиком новых заявок.
- Администратор ведет заявку по статусам: `pending` -> `processing` -> `invoice_sent` -> `paid` -> `completed`.
- На шаге `invoice_sent` администратор указывает сумму счета и комментарий. Ограничения сверху по сумме нет: заявки больше 1 000 000 ₽ обрабатываются обычным образом.
- После статуса `paid` кнопка "Зачислить на баланс" переводит заявку в `completed`, увеличивает баланс пользователя и пишет запись в `transactions` с email администратора.
- Быстрая ручная корректировка баланса в `/admin` также пишет аудит в `transactions`.
- Оплата с баланса списывает деньги и создает оплаченный заказ.
- Оплата картой создает заказ со статусом `awaiting_payment`; менеджер связывается с клиентом.

## Как добавить новую услугу

1. Добавьте изображение в `public/services/`.
2. Откройте `lib/services.ts`.
3. Добавьте объект в массив `services` с уникальными `id` и `slug`.
4. Используйте категорию из `serviceCategories`.

В `lib/services.ts` есть комментарий с требованиями к реальным фото: вентиляционные установки, воздуховоды, чиллеры, фанкойлы, автоматика, холодильные контуры и технические помещения.
